Reservoir Model¶

class morl4water.core.models.reservoir.Reservoir(name: str, max_capacity: float, max_action: list[float], objective_function, integration_timestep_size: relativedelta, evap_rates: list[float], evap_rates_timestep_size: relativedelta, storage_to_minmax_rel: list[list[float]], storage_to_level_rel: list[list[float]], storage_to_surface_rel: list[list[float]], objective_name: str = '', stored_water: float = 0, spillage: float = 0, observation_space=Box(0.0, 1.0, (1,), float32), action_space=Box(0.0, 1.0, (1,), float32))¶

Bases: ControlledFacility

A class representing a reservoir within a water management system.

This class is used to model the storage, release, and evaporation processes of a reservoir, including the management of water levels, inflows, outflows, and associated evaporation. The reservoir interacts with the surrounding environment and is controlled by various parameters such as release rates, storage, and evaporation rates.

determine_info() → dict¶

Returns information about the current state of the reservoir.

This includes details such as the current stored water volume, water level, release, and evaporation rates.

Returns:

A dictionary containing key information about the reservoir (e.g., name, stored water,

current level, current release, evaporation rates).

Return type:

dict

determine_observation() → float¶

Retrieves the current observation of the reservoir, which is the amount of stored water.

Returns:

The current stored water in the reservoir (in m³). If no water is stored, returns 0.

Return type:

float

determine_outflow(actions: array) → list[float]¶

Determines the average monthly water release from the reservoir based on the given actions.

The release is adjusted according to the current storage level, evaporation rates, and actions taken.

Parameters:

actions (np.array) – Array of release actions that control the reservoir’s water release.

Returns:

The average release of water per timestep.

Return type:

list[float]

determine_reward() → float¶

Calculates the reward for the reservoir based on its current stored water level.

This method uses the objective function to evaluate the reward based on the water level, which is derived from the stored water volume.

Returns:

The calculated reward for the reservoir.

Return type:

float

determine_time_idx() → int¶

Determines the index for the evaporation rate based on the current date and timestep.

The function maps the current date to the appropriate index in the evaporation rate array, considering the timestep size (months, days, or hours).

Returns:

The index corresponding to the current date in the evaporation rate array.

Return type:

int

Raises:

ValueError – If the timestep size is unsupported (i.e., not months, days, or hours).

is_terminated() → bool¶

Checks if the reservoir’s simulation has reached a termination condition.

The simulation is terminated if the stored water exceeds the maximum capacity or falls below 0.

Returns:

True if the simulation should be terminated (due to overflow or depletion), otherwise False.

Return type:

bool

level_to_minmax(h) → tuple[ndarray, ndarray]¶

Calculates the minimum and maximum possible release rates based on the given water level.

h (float) – The water level (height) to calculate the release rates for.

Returns:

A tuple containing two arrays - the minimum and maximum release rates.

tuple

static modified_interp(x: float, xp: float, fp: float, left=None, right=None) → float¶

Performs linear interpolation between two points with custom handling for out-of-bound values.

Parameters:

  • x (float) – The input value to interpolate.

  • xp (float) – The known input values for interpolation.

  • fp (float) – The corresponding output values for interpolation.

  • left (optional) – The value to return if x is less than the smallest value in xp.

  • right (optional) – The value to return if x is greater than the largest value in xp.

Returns:

The interpolated value based on the provided input values.

Return type:

float

reset() → None¶

Resets the state of the reservoir to its initial conditions for a new simulation.

This method resets the storage, release, and level vectors, and restores the initial stored water volume.
